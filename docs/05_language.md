# Language

SWMのLanguageモジュールは、自然言語として完全な架空言語を作成するためのものではない。
作品中の人名、地名、民族名、国名、神名、称号などに一貫したフレーバーを与えるため、命名処理に必要な語根、音形、最小限の文法情報を扱う。

## 目的

- 祖語・派生言語・他言語からの影響を表現する。
- 人名・地名・国名・民族名・神名・称号などの命名に使用する。
- 漢字などの意味IDから、命名用の語根を生成する。
- 音形を安定したASCII文字列として扱う。
- 格・数・助詞・前置詞・語順など、命名に必要な文法フレーバーを扱う。

## Language Seed System

SWMのLanguage Seed Systemは、漢字などの意味IDを起点に、命名用の語根を生成する仕組みである。

基本的な流れは次の通り。

```text
意味ID
  ↓
実在言語の訳語または読み
  ↓
sound_key
  ↓
機械的加工
  ↓
祖語語根
  ↓
複合語・人名・地名・国名など
```

漢字を意味IDとして使う場合、漢字1文字に対して1つの語または読みを対応させる。
翻訳精度は厳密には求めず、命名用語根を得るための意味的手がかりとして扱う。

## 語彙データ

語彙素材は、言語ごとにCSVで管理する。

```csv
char,word
山,shan1
川,chuan1
```

- `char` は意味IDを表す。
- `word` は入力語、読み、または `sound_key` を表す。
- 1ファイルは原則として1言語または1入力体系に対応する。
- UTF-8 CSVはBOM付き、改行文字はCRLFとする。

## sound_key

`sound_key` は、各言語の語をSWMの命名処理に利用するためのASCII表記である。
これはIPAやX-SAMPAの代替を目的とした厳密な音声表記ではない。
作品世界の命名処理で区別したい音形を、安定した文字列として扱うための独自表記体系である。

### 基本方針

- 英小文字を基本的な音形本体として扱う。
- 英小文字以外のASCII文字は装飾として扱う。
- 数字は声調またはピッチなどの韻律情報として扱う。
- 同一の音形または同一視する音形に対して、複数の標準表記を与えない。
- 見た目の簡潔さよりも、機械処理上の一貫性を優先する。
- 対象言語または素材体系において対立しない音声差は、原則として代表表記に丸める。
- 命名処理上区別する必要がある音声差は、装飾記号を用いて区別する。

## 声調

声調を持つ言語では、声調を数字で表す。
数字は音素ではなく装飾情報として扱い、直前の音節に付加する。

数字が現れた位置を音節末尾とみなすため、音節境界を表す専用記号は使用しない。

例:

```text
shan1
wang2
huo3
yue4
bei3jing1
guang3zhou1
```

## 子音

### 基本子音

基本的な子音は英小文字で表す。

```text
p b
t d
k g
m n
s z
h
r l
w y
```

`q` と `x` は、ピンイン式の値には固定しない。
SWM標準では、必要に応じて次のような汎用的な子音に割り当てるために予約する。

```text
q = /q/ 系、口蓋垂・奥寄りの k 系
x = /x/ 系、軟口蓋・喉寄りの摩擦音系
```

### 有気音

`h` は有気音マーカーとして後置できる。

```text
p  = p 系
ph = 有気 p 系

t  = t 系
th = 有気 t 系

k  = k 系
kh = 有気 k 系

ts  = ts 系
tsh = 有気 ts 系
```

`h` が単独で現れる場合は、/h/ 系の子音として扱う。
後置された `h` は有気音マーカーとして解釈できるため、単独の `h` とは区別できる。

### sh 系摩擦音

/ʃ/, /ɕ/, /ʂ/ などの sh 系摩擦音は、対立が不要な場合は `s'` で代表させる。

```text
s  = /s/ 系
s' = sh 系代表音
```

対立を保持する必要がある場合のみ、次のように装飾を分ける。

```text
s' = /ʃ/ 系
s^ = /ɕ/ 系
s` = /ʂ/ 系
```

### ch 系破擦音

対応する破擦音は `ts` 系で表す。
対立が不要な場合は `ts'` で代表させる。

```text
ts  = /ts/ 系
ts' = ch 系代表音
```

対立を保持する必要がある場合のみ、次のように装飾を分ける。

```text
ts' = /tʃ/ 系
ts^ = /tɕ/ 系
ts` = /tʂ/ 系
```

有気音は `h` を後置する。

```text
tsh   = /tsʰ/ 系
ts'h  = /tʃʰ/ 系
ts^h  = /tɕʰ/ 系
ts`h  = /tʂʰ/ 系
```

### ピンインからの子音変換例

ピンインは `sound_key` そのものではなく、入力表記の一種として扱う。
ピンインからSWM標準 `sound_key` に変換する場合、次のような対応を用いる。

```text
b  -> p
p  -> ph

d  -> t
t  -> th

g  -> k
k  -> kh

z  -> ts
c  -> tsh
s  -> s

j  -> ts^
q  -> ts^h
x  -> s^

zh -> ts`
ch -> ts`h
sh -> s`
r  -> z`
```

例:

```text
qi1     -> ts^hi1
xia4    -> s^ia4
jing1   -> ts^ing1
zhong1  -> ts`ong1
chang2  -> ts`hang2
shan1   -> s`an1
cao3    -> tshao3
```

## 母音

### 基本母音

母音は `a, e, i, o, u` の5母音を基本として表す。

```text
a e i o u
```

対象言語または素材体系に母音の対立がない場合、近い基本母音に丸める。
対立を保持する必要がある場合のみ、英小文字以外の装飾記号を後置して区別する。

### 代表的な母音装飾

```text
: = 長母音
~ = 鼻母音
' = 前寄り・開き気味
` = 後ろ寄り
^ = 中央寄り
" = 中舌化・前舌円唇化など
@ = 曖昧母音
- = 非円唇化
```

### 母音表記例

```text
a   = /a/ 系
a'  = /æ/ 系
a^  = /ɐ/ 系
a"  = /ä/ 系
a`  = /ɑ/ 系
a&  = /ɶ/ 系

e   = /e/ 系
e'  = /ɛ/ 系
e@  = /ə/ 系

i   = /i/ 系
i'  = /ɪ/ 系

o   = /o/ 系
o'  = /œ/ 系
o^  = /ɞ/ 系
o"  = /ø/ 系
o`  = /ɒ/ 系

u   = /u/ 系
u'  = /ʊ/ 系
u"  = /y/ 系
u-  = /ɯ/ 系
```

`a&` はかなり特殊な母音を扱うための予約に近い表記であり、通常は必要になった場合のみ使用する。

### 長母音・鼻母音

長母音は `:` を後置する。

```text
a:
e:
i:
o:
u:
```

鼻母音は `~` を後置する。

```text
a~
e~
i~
o~
u~
```

声調を持つ長母音や鼻母音では、母音装飾の後に声調数字を置く。

```text
ma:1
ma~2
```

## 日本語由来のsound_key

日本語由来の `sound_key` では、は行を `h` 系として扱う。
そのため「ふ」は `hu` と表記し、「ふぁ・ふぃ・ふぇ・ふぉ」は `hwa, hwi, hwe, hwo` と表記する。

```text
は = ha
ひ = hi
ふ = hu
へ = he
ほ = ho

ふぁ = hwa
ふぃ = hwi
ふぇ = hwe
ふぉ = hwo
```

`fa, fi, fe, fo` は、日本語の「ふぁ」ではなく、/f/ 系子音を持つ別言語由来の音形に使用できる。

日本語の母音の無声化、鼻濁音、撥音の細かな同化、`hi` の実音 [çi]、`hu` の実音 [ɸɯ] などは、原則として `sound_key` には反映しない。

ただし、命名処理上区別したい音形は保持する。

```text
シ  = s'i
スィ = si

チ  = ts'i
ティ = ti

ツ  = tsu
トゥ = tu

ジ  = z'i
ズィ = zi
```

## 文法フレーバー

SWMのLanguageモジュールでは、自然言語として自由に会話できる文法体系ではなく、名前や短い語句にそれらしい構造を与えるための最小限の文法フレーバーを扱う。

対象になり得る要素は次の通り。

- 数
- 格
- 所有表現
- 助詞
- 前置詞
- 後置詞
- 語順
- 文法上の性
- 名詞クラス
- 冠詞風要素

祖語・古語・部族語・神官語では、格・数・名詞クラスなどをやや複雑にしてもよい。
一方、帝国共通語・交易語・行政語などでは、文法カテゴリを整理し、語順や前置詞・後置詞で補う形に単純化できる。

## 実装上の段階

Language Seed Systemは、次の段階に分けて実装する。

```text
Stage 1: Glyph Mapping
  意味IDから入力語または読みを得る。

Stage 2: Sound Normalization
  入力語または読みをSWM標準sound_keyに変換する。

Stage 3: Proto Form
  sound_keyを機械加工して祖語形を作る。

Stage 4: Compound Word
  複数の意味IDから複合語を作る。

Stage 5: Name Form
  複合語を人名・地名・国名などに整形する。
```
